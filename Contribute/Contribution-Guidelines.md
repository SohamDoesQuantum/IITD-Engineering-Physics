# CONTRIBUTING

## Submission Process
1. Fill the contribution form 
2. Upload only files you have rights to share.
3. Follow the naming and folder rules below.

---

## Accepted Contributions
- Notes, assignments, tutorials, lab manuals  
- PYQs and solutions  
- Project reports, code, simulations  
- Internship and research experiences  
- Higher-studies application experiences  
- Recommended books, tools, links, GitHub repos  

---

## File Naming Convention
```
CourseCode_Name_Type_Year.ext
```
Examples:
```
EP203_QM1_Notes_2024.pdf
EP202_EM_Assignment3_Solutions.pdf
EP120_Mech_LabManual.pdf
```

---

## Folder Structure
```
Academics/
└── YearX/
    └── CourseName-CourseCode/
        ├── Notes/
        ├── Assignments/
        ├── PYQs/
        ├── Projects/
        └── Extra/
```

---

## Rules
- No copyrighted textbook pages  
- Remove personal details from PDFs  
- Keep files clean, readable, properly named  
- Avoid uploading files >250MB  
- Only upload original or permission-granted content  

---

## GitHub Contribution Steps
```
1. Fork the repository
2. Create a new branch
3. Add files to the correct folder
4. Commit with a clear message
5. Open a Pull Request
```

---

## Attribution
Contributors are credited unless they choose anonymity.  
Attribution options:
- Full name  
- GitHub username  
- Anonymous
